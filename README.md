# 🛋️ دمياط للأثاث - Damyat Furniture

![Damyat Furniture](https://images.unsplash.com/photo-1618220179428-22790b461013?w=1200&h=400&fit=crop&q=80)

منصة تجارة إلكترونية متكاملة لبيع الأثاث الفاخر، مبنية بأحدث تقنيات الويب مع تصميم عصري (Dark Luxury) وتجربة مستخدم لا مثيل لها. يوفر المتجر تصفح المنتجات، تخصيص مقاسات الأثاث، وإتمام الطلبات، مع لوحة تحكم خلفية قوية مدعومة بقاعدة بيانات سحابية.

---

## ✨ المميزات الرئيسية

- **🎨 تصميم فاخر وجذاب**: واجهة مستخدم (UI) تعتمد على الـ Glassmorphism والألوان الذهبية الدافئة.
- **⚡ أداء فائق**: مبني باستخدام Vite و React لضمان أسرع وقت تحميل.
- **🛒 سلة تسوق متقدمة**: نظام سلة تسوق ذكي يدعم "باقات الغرف" وتخصيص المقاسات لكل قطعة أثاث (Zustand).
- **💳 بوابة دفع متكاملة**: صفحة إتمام الطلب بتصميم متعدد الخطوات يدعم الشحن واختيار طرق الدفع.
- **🎬 حركات وتأثيرات بصرية**: تجربة تصفح تفاعلية سلسة بفضل `framer-motion`.
- **🗄️ قاعدة بيانات سحابية**: متصل بقاعدة بيانات **Neon (PostgreSQL)** لحفظ المنتجات، الطلبات، ورسائل التواصل.
- **🚀 جاهز للنشر (Deployment Ready)**: معد خصيصاً ليتم نشره بضغطة زر على منصة **Vercel** كـ (Fullstack App).

---

## 🛠️ التقنيات المستخدمة

### الواجهة الأمامية (Frontend)
- **React.js 18** (Vite)
- **Tailwind CSS** (للتصميم المتجاوب السريع)
- **Framer Motion** (للحركات والانتقالات)
- **Zustand** (لإدارة حالة التطبيق والسلة)
- **React Router Dom** (للتنقل بين الصفحات)
- **Lucide React** (للأيقونات الحديثة)
- **Axios** (لجلب البيانات من الـ API)

### الخادم (Backend)
- **Node.js & Express.js** (لإنشاء خادم سريع)
- **PostgreSQL (Neon)** (قاعدة بيانات Serverless)
- **pg** (للربط المباشر مع قاعدة البيانات)
- **dotenv** & **cors** (لإدارة البيئة والحماية)

## 📦 خطوات التشغيل المحلي (Local Setup)

### 1. استنساخ المشروع
```bash
git clone <repository-url>
cd damyat-furniture
```

### 2. تثبيت المكتبات
```bash
npm install
```

### 3. إعداد البيئة (Environment Variables)
قم بإنشاء ملف `.env` في المسار الرئيسي للمشروع وضع فيه بيانات الاتصال:
```env
VITE_API_URL=http://localhost:5000/api
VITE_APP_NAME=دمياط للأثاث
DATABASE_URL="postgresql://<user>:<password>@<host>/<database>?sslmode=require"
PORT=5000
```

### 4. تشغيل المشروع (Frontend + Backend)
تم إعداد أمر واحد لتشغيل واجهة الموقع والخادم معاً باستخدام `concurrently`:
```bash
npm run dev:all
```
- واجهة الموقع ستعمل على: `http://localhost:5173`
- الخادم (API) سيعمل على: `http://localhost:5000`

## 🚀 النشر على Vercel (Deployment)

هذا المشروع مُجهز ليتم رفعه مباشرة إلى **Vercel** ليعمل كـ Fullstack Application بفضل ملف `vercel.json` الموجود مسبقاً.

### الخطوات:
1. ارفع المشروع إلى حسابك في **GitHub**.
2. اذهب إلى منصة [Vercel](https://vercel.com) وقم بتسجيل الدخول.
3. اضغط على **Add New Project** واختر مستودع المشروع من GitHub.
4. في قسم **Environment Variables**، قم بإضافة المتغير التالي:
   - الاسم: `DATABASE_URL`
   - القيمة: رابط قاعدة بيانات Neon الخاص بك.
5. اضغط على **Deploy**.

> **ملاحظة:** Vercel سيقوم تلقائياً ببناء واجهة React وتشغيل ملفات `server/index.js` كـ Serverless Functions، فلا تحتاج لأي إعدادات إضافية!
